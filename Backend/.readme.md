# 🎬 Movie Platform — Backend API

A production-ready REST API built with **Node.js**, **Express.js**, and **MongoDB**. Supports JWT authentication, favorites management, and watch history tracking.

---

## 📁 Project Structure

```
Backend/
├── server.js                     # Entry point
├── .env.example                  # Environment variable template
├── package.json
└── src/
    ├── app.js                    # Express app setup
    ├── config/
    │   └── database.js           # MongoDB connection
    ├── models/
    │   └── user.model.js         # User schema (with favorites & history)
    ├── controllers/
    │   ├── auth.controller.js    # Signup, login, logout, getMe
    │   ├── favorites.controller.js
    │   └── watchHistory.controller.js
    ├── routes/
    │   ├── auth.routes.js        # /api/auth/*
    │   └── movie.routes.js       # /api/movies/*
    └── middlewares/
        ├── auth.middleware.js    # JWT protect middleware
        └── error.middleware.js   # Global error handler
```

## ⚙️ Setup

### 1. Install dependencies

### 2. Configure environment
```bash
cp .env.example .env
```

## 📡 API Endpoints

### Auth Routes — `/api/auth`

| Method | Endpoint         | Access  | Description              |
|--------|-----------------|---------|--------------------------|
| POST   | `/signup`       | Public  | Register new user         |
| POST   | `/login`        | Public  | Login & receive JWT token |
| POST   | `/logout`       | Private | Clear auth cookie         |
| GET    | `/me`           | Private | Get logged-in user info   |

### Movie Routes — `/api/movies` *(All Protected)*

#### Favorites
| Method | Endpoint                  | Description                     |
|--------|---------------------------|---------------------------------|
| GET    | `/favorites`              | Get all favorite movies         |
| POST   | `/favorites`              | Add a movie to favorites        |
| GET    | `/favorites/:movieId`     | Check if movie is in favorites  |
| DELETE | `/favorites/:movieId`     | Remove movie from favorites     |

#### Watch History
| Method | Endpoint                  | Description                     |
|--------|---------------------------|---------------------------------|
| GET    | `/history`                | Get full watch history          |
| POST   | `/history`                | Add/update movie in history     |
| DELETE | `/history`                | Clear all watch history         |
| DELETE | `/history/:movieId`       | Remove specific movie from history |

## 🔐 Authentication

- JWT token is returned on login/signup in both the **response body** and as an **httpOnly cookie**
- Protected routes accept token via:
  - `Authorization: Bearer <token>` header
  - `token` cookie (automatic if using cookie-based auth)
- Passwords are hashed using **bcryptjs** (salt rounds: 12)
- Tokens expire after **7 days** by default

---

## 🛡️ Security Features

- Passwords never returned in API responses (`select: false`)
- httpOnly cookies prevent XSS attacks
- CORS configured for specific frontend origin
- Global error handler prevents stack trace leakage in production
- Mongoose duplicate key errors return friendly messages
